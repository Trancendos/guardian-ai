# Security Policy

## Supported Versions

This repository follows an **N-to-N-1 dependency policy**:

- **N** = latest stable upstream release
- **N-1** = one major version behind latest

Any dependency that is older than N-1 is treated as non-compliant and must be upgraded.

## Vulnerability Reporting

Please report security vulnerabilities privately through GitHub Security Advisories.
Do not open public issues for unpatched vulnerabilities.

## Security Controls in This Repository

- **Dependabot** for weekly dependency and GitHub Actions updates.
- **Dependency Review** on pull requests to block high-risk dependency changes.
- **npm audit** in CI for known vulnerabilities (HIGH+ threshold).
- **Trivy filesystem scan** for HIGH/CRITICAL vulnerabilities.
- **CodeQL** scheduled and on-change static analysis.
- **CycloneDX SBOM** artifact generated by security workflow.
- **N-to-N-1 policy check** (`scripts/check-n-policy.mjs`) to prevent version drift.

## Remediation Targets

- **Critical CVE**: patch/mitigate within 24 hours.
- **High CVE**: patch/mitigate within 3 business days.
- **Medium CVE**: patch/mitigate within 14 days.
- **Low CVE**: patch during routine maintenance.

## Operational Guidance

1. Keep `main` protected and require passing CI + security workflows.
2. Treat failed security jobs as release blockers.
3. Review and merge dependency updates weekly.
4. Regenerate SBOM after major dependency updates.
